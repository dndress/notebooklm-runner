## How to use this visual guide

Use this file only as a character reference and visual style reference. Do not illustrate this guide, its headings, its rules, player names, Discord IDs, labels, or notes.

The full visual description is the canonical source for each character’s appearance. Never contradict it.

The Short Character Lock is a compact reminder for panel prompts. It does not replace the full description.

When a named character is the main focus of a panel, use the full visual description to preserve detail.

When multiple named characters appear in one panel, use each character’s Short Character Lock to keep them distinct and avoid mixing traits.

When a panel names a character, do not replace that character with a generic fantasy figure or another character from the guide.

If a character has “Important distinction” or “Do not confuse with” notes, follow them strictly.

For final comic pages, include only the story narration text. Do not print visual instructions, character locks, guide notes, labels, player names, Discord IDs, or production notes.

## Final PDF visual style guide

Use this section only to guide the final comic PDF. Do not illustrate or print this section, its heading, its rules, labels, file names, or instruction text.

Create a dark fantasy graphic novel comic based on the session outline. This is a graphic novel, not a presentation deck. Do not add slide titles, bullet points, presenter-style layouts, decorative headings, icons, or presentation elements.

Use the session outline as the main source for page order, panel order, visual action, required visible characters, required subjects, and narration text. Use this character visual guide as the canonical source for the appearance of every named player character and named companion. Do not invent or change a named character’s gender, ancestry, hair, clothing, weapons, class identity, companion, magic style, or main visual anchors.

Only print the story narration inside the comic panels. Use only the text after `Narration:` from the session outline. Do not print labels such as `Visual:`, `Narration:`, `Required visible characters:`, `Required visible subjects/enemies:`, `Optional visible characters/subjects:`, `Panel importance:`, `Short character lock:`, `Production note:`, or any other guide text.

Style: dark fantasy horror graphic novel illustration with mature comic-book aesthetics, neo-noir lighting, heavy black ink shadows, sharp readable linework, limited color palette, and high contrast. Use deep blacks, cold teal-blue shadows, muted parchment tones, restrained crimson-orange supernatural glow, and occasional blue-white magical light.

The pages should feel ominous, nocturnal, serious, adult-targeted, and story-driven. Use strong silhouettes, dramatic single focal points, cinematic cropping, screenprint-like color blocking, and controlled gritty texture. Avoid cute, comedic, anime, photorealistic, 3D-rendered, glossy, or overly clean fantasy art.

Use texture carefully. Add grain, rough ink, smoke, cracks, sparks, bloodless impact marks, weathered metal, worn paper texture, and firelight texture only where they support the focal scene. Avoid muddy rendering, excessive scratch marks, noisy grunge overlays, over-hatching, and deep-fried texture.

Page layout: use a full-bleed dark graphic novel page layout. The entire page should feel illustrated, atmospheric, or intentionally inked. Do not leave plain white gutters, white margins, empty white wedges, or unused presentation-like space between panels. If gutters or borders are visible, make them black, dark, textured, painted, or integrated into the page atmosphere.

Panels may be irregular, tilted, overlapping, or separated by dark ink gutters, but the page must feel like one complete comic composition. Avoid clean white slide backgrounds, floating panels on white space, diamond inset panels that create blank white corners, or collage layouts with unused empty areas. Narration boxes may use parchment tones, but the surrounding page should remain dark and story-focused.

Use dynamic graphic novel panel hierarchy. Do not make all panels the same size or arrange them as an even grid unless the outline specifically calls for a calm, orderly moment. Larger story beats, reveals, attacks, monster moments, character entrances, rescues, major discoveries, and cliffhangers should receive larger panels. Smaller panels should be used for reactions, details, transitions, scouting, or supporting actions.

Characters, weapons, magic, smoke, debris, shadows, monsters, or companions may break past panel borders when it improves the story. Panel-breaking elements should not hide the required characters or make the reading order unclear.

Use clear Western comic reading order. The page must read from top-left to bottom-right, following a natural Z-pattern. The first panel should be visually placed near the upper-left area, and the final panel or cliffhanger should land near the lower-right area unless the outline clearly requires a different structure. Dynamic layouts are allowed, but the panel sequence and narration box placement must stay obvious. Do not create right-to-left, manga-style, bottom-to-top, or scattered ambiguous layouts.

Every panel should prioritize story clarity. If a panel names a required visible character, that character must appear and must be visually identifiable using this visual guide. If a panel names only a subject, object, NPC, monster, or location, it is acceptable to show that subject without showing the heroes.
